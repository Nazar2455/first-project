Практика: минимальный бот на aiogram 3

main.py — стартовая версия
import asyncio
import os
from aiogram import Bot, Dispatcher
from aiogram.filters import Command
from aiogram.types import Message
from dotenv import load_dotenv

load_dotenv()
TOKEN = os.getenv("TELEGRAM_BOT_TOKEN")

dp = Dispatcher()

@dp.message(Command("start"))
async def cmd_start(message: Message):
    await message.answer("Привет! Я lead-бот. Нажми /help")

@dp.message(Command("help"))
async def cmd_help(message: Message):
    await message.answer("Команды: /start /help")

async def main():
    bot = Bot(TOKEN)
    await dp.start_polling(bot)

if __name__ == "__main__":
    asyncio.run(main())
Разбор: load_dotenv() грузит токен из .env, Dispatcher регистрирует хендлеры, start_polling запускает цикл получения сообщений.

Практика: базовый парсер + CSV

main.py — пример парсинга карточек
import csv
import requests
from bs4 import BeautifulSoup

URL = "https://example.com/catalog"
response = requests.get(URL, timeout=20)
response.raise_for_status()

soup = BeautifulSoup(response.text, "html.parser")
cards = soup.select(".product-card")

rows = []
for card in cards[:100]:
    title = card.select_one(".title").get_text(strip=True)
    price = card.select_one(".price").get_text(strip=True)
    rows.append({"title": title, "price": price})

with open("result.csv", "w", newline="", encoding="utf-8") as f:
    writer = csv.DictWriter(f, fieldnames=["title", "price"])
    writer.writeheader()
    writer.writerows(rows)

print(f"Сохранено: {len(rows)}")
Разбор: raise_for_status() сразу ловит HTTP-ошибки, select/select_one — CSS-селекторы, CSV нужен для быстрого показа результата клиенту.

FSM-форма лидов (имя → телефон → комментарий)

states/lead_state.py
from aiogram.fsm.state import StatesGroup, State

class LeadForm(StatesGroup):
    name = State()
    phone = State()
    comment = State()
handlers/lead.py — фрагмент
@router.message(Command("lead"))
async def lead_start(message: Message, state: FSMContext):
    await state.set_state(LeadForm.name)
    await message.answer("Как вас зовут?")

@router.message(LeadForm.name)
async def lead_name(message: Message, state: FSMContext):
    await state.update_data(name=message.text)
    await state.set_state(LeadForm.phone)
    await message.answer("Введите телефон")
Смысл FSM: ты чётко управляешь диалогом. Это фундамент всех коммерческих ботов, где нужна форма заявки.

SQLite для сохранения лидов

db/database.py
import aiosqlite

CREATE_SQL = """
CREATE TABLE IF NOT EXISTS leads (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER,
    name TEXT,
    phone TEXT,
    comment TEXT,
    created_at TEXT DEFAULT CURRENT_TIMESTAMP
);
"""

async def init_db(path: str = "bot.db"):
    async with aiosqlite.connect(path) as db:
        await db.execute(CREATE_SQL)
        await db.commit()

Google Sheets экспорт для парсера

services/sheets.py
import gspread
from google.oauth2.service_account import Credentials

SCOPES = ["https://www.googleapis.com/auth/spreadsheets"]

def get_sheet(creds_path: str, table_name: str):
    creds = Credentials.from_service_account_file(creds_path, scopes=SCOPES)
    client = gspread.authorize(creds)
    return client.open(table_name).sheet1

def append_rows(sheet, rows):
    if rows:
        sheet.append_rows(rows, value_input_option="USER_ENTERED")
Почему клиентам это нравится: им не нужно открывать код, всё видно в таблице в реальном времени.

Устойчивость к ошибкам сайта

Retry-обёртка запросов
import time
import requests

def safe_get(url: str, retries: int = 3, timeout: int = 20):
    for attempt in range(1, retries + 1):
        try:
            response = requests.get(url, timeout=timeout)
            response.raise_for_status()
            return response
        except Exception as e:
            if attempt == retries:
                raise
            time.sleep(1.5 * attempt)
Это спасает от временных падений сайта и уменьшает «ложные» ошибки у клиента.

💰 Продажи: как конвертировать навык в оплату

Мини-скрипт созвона/чата

  1. Зафиксируй бизнес-цель клиента (что должно улучшиться).
  2. Скажи, что именно автоматизируешь.
  3. Покажи похожий кейс + цифру результата.
  4. Дай дедлайн и 2 варианта стоимости.
  5. Согласуй ближайший следующий шаг (ТЗ/прототип).

Почему ты можешь брать дороже

Потому что продаёшь не “скрипт”, а результат: автоматизацию, экономию времени, прозрачность данных, поддержку.

✅ Универсальный чеклист качества перед сдачей

  • Код запускается с чистой установки зависимостей
  • Есть файл .env.example
  • README содержит запуск шаг за шагом
  • Логи ошибок не молчат, а объясняют проблему
  • Видео-демо: 30–90 секунд реального сценария
  • Нет токенов и ключей в репозитории
Если этот блок соблюдён, вероятность возвратов и правок от клиента падает в разы.

🏁 Финальный маршрут на 30 дней (как проходить по этому курсу)

  1. Каждый день открываешь основной план + эту страницу.
  2. Берёшь задачи дня, затем в этом курсе читаешь соответствующий модуль.
  3. Копируешь код-шаблон, адаптируешь под свой проект.
  4. Делаешь коммит, демо, отправляешь отклики.
  5. Отмечаешь чекбоксы прогресса (они сохраняются локально).

Главный принцип: Лучше 80% готового и опубликованного, чем 100% идеального, но не проданного.
