🕷️ ПАРСИНГ: delivery gate + client summary

Цель: внедрить критерии «готово к сдаче» перед передачей результата клиенту.

Связь с теорией: используешь quality/duration-пороги и формируешь понятный summary для клиента.

Фокус на смысл: сдача должна быть управляемой: либо ready, либо blocked с прозрачной причиной.

Что нужно сделать

  1. Есть пороги quality/duration.
  2. Есть status code ready/blocked.
  3. Есть client_summary с KPI.
  4. Есть финальное решение по сдаче.

Definition of Done

  • Gate стабильно определяет статус сдачи.
  • Summary содержит ключевые KPI по прогону.
  • При блокировке есть чёткая причина.
