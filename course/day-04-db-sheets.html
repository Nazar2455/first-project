Модуль 4 · День 4: хранение данных и облачная выгрузка

Сегодня ты переводишь проекты из «демо» в «полезный сервис»: бот сохраняет заявки в БД, парсер отдаёт результат в Google Sheets.

🗺️ Как пройти День 4 (без перегруза)

  1. Сначала маленькие куски — SQL и Sheets по 3-8 строк.
  2. Потом полный модуль — видишь, как части работают вместе.
  3. Потом интеграция — подключаешь к своему боту и парсеру.
  4. Потом проверка — заявка реально в БД, строки реально в таблице.
Критерий «день закрыт»: не «я прочитал», а «я сохранил данные и могу их показать».

🤖 БОТЫ: зачем SQLite в боте

Если не сохранять заявки — они исчезают после перезапуска. SQLite решает это без сложной настройки сервера.

Проще говоря: SQLite — это твоя «память» проекта. Без неё бот отвечает, но ничего не накапливает.

Результат дня по ботам

  • Таблица leads создаётся автоматически.
  • Форма из Дня 3 пишет данные в БД.
  • Можно получить последние заявки для админки.

🧪 БОТЫ: мини-примеры SQL/SQLite

Пример 1: CREATE TABLE
CREATE TABLE IF NOT EXISTS leads (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER,
    name TEXT,
    phone TEXT,
    comment TEXT,
    created_at TEXT DEFAULT CURRENT_TIMESTAMP
);
Пример 2: INSERT
INSERT INTO leads (user_id, name, phone, comment)
VALUES (?, ?, ?, ?);

Важно: ? — защита от SQL-инъекций и корректная подстановка значений.

Мини-пример 3: получить последние записи

SELECT + ORDER + LIMIT
SELECT id, name, phone
FROM leads
ORDER BY id DESC
LIMIT 5;

Так обычно строят быстрый «админский просмотр» свежих заявок.

Мини-пример 4: commit после записи

Почему без commit данные не видны
await db.execute(query, params)
await db.commit()  # фиксируем изменения в файле БД

Без commit() запись может не сохраниться физически, особенно заметно после перезапуска.

🤖 БОТЫ: полный рабочий пример с комментариями

db/database.py
import aiosqlite

DB_PATH = "bot.db"

CREATE_LEADS_TABLE = """
CREATE TABLE IF NOT EXISTS leads (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER,
    name TEXT,
    phone TEXT,
    comment TEXT,
    created_at TEXT DEFAULT CURRENT_TIMESTAMP
);
"""

async def init_db() -> None:
    # Создаём файл БД и таблицу при старте приложения
    async with aiosqlite.connect(DB_PATH) as db:
        await db.execute(CREATE_LEADS_TABLE)
        await db.commit()
db/repository.py
import aiosqlite
from db.database import DB_PATH

async def save_lead(user_id: int, name: str, phone: str, comment: str) -> None:
    query = """
    INSERT INTO leads (user_id, name, phone, comment)
    VALUES (?, ?, ?, ?)
    """
    async with aiosqlite.connect(DB_PATH) as db:
        # Пишем новую заявку
        await db.execute(query, (user_id, name, phone, comment))
        await db.commit()

async def get_last_leads(limit: int = 10) -> list[tuple]:
    query = """
    SELECT id, name, phone, comment, created_at
    FROM leads
    ORDER BY id DESC
    LIMIT ?
    """
    async with aiosqlite.connect(DB_PATH) as db:
        cursor = await db.execute(query, (limit,))
        rows = await cursor.fetchall()
        return rows
Фрагмент из FSM завершения (интеграция)
from db.repository import save_lead

# ... внутри финального шага FSM
await save_lead(
    user_id=message.from_user.id,
    name=data["name"],
    phone=data["phone"],
    comment=data["comment"]
)
# После этого заявка сохранена в SQLite

Пошаговая карта потока данных в боте

  1. Пользователь завершает FSM-форму.
  2. Ты берёшь данные из state.get_data().
  3. Вызываешь save_lead(...) и фиксируешь commit.
  4. Пользователь получает подтверждение, админ может увидеть заявку позже.
Это уже полноценный сценарий «сбор и хранение лидов», который можно показывать в портфолио как бизнес-функцию.

🧯 БОТЫ: частые ошибки при работе с SQLite

  • Таблица не найдена: init_db() не вызван при старте приложения.
  • Записи не появляются: забыт await db.commit().
  • Пустые поля: на одном из шагов FSM не сохранены данные в state.
  • Ошибки импорта: модуль aiosqlite не установлен в активном окружении.

🕷️ ПАРСИНГ: зачем Google Sheets

Google Sheets — это «панель для клиента»: он видит результат без запуска кода, можно делиться ссылкой и хранить историю.

То есть ты превращаешь локальный CSV в «живой отчёт», который открывается по ссылке с телефона или ноутбука.

Результат дня по парсингу

  • Подключён сервисный аккаунт Google.
  • Парсер дописывает строки в таблицу.
  • Отчёт доступен в облаке.

🧪 ПАРСИНГ: мини-примеры Sheets API

Пример 1: авторизация
import gspread
from google.oauth2.service_account import Credentials

SCOPES = ["https://www.googleapis.com/auth/spreadsheets"]
creds = Credentials.from_service_account_file("service-account.json", scopes=SCOPES)
client = gspread.authorize(creds)
Пример 2: добавить одну строку
sheet = client.open("Parser Results").sheet1
sheet.append_row(["Название", "Цена", "Ссылка"])  # Одна строка

Важно: сначала поделись таблицей с email сервисного аккаунта (из JSON-ключа), иначе будет ошибка доступа.

Мини-пример 3: пакетная запись (быстрее)

append_rows вместо append_row в цикле
prepared = [
    ["Товар A", "12990", "https://example.com/a"],
    ["Товар B", "15990", "https://example.com/b"],
]
sheet.append_rows(prepared, value_input_option="USER_ENTERED")

Когда строк много, пачечная запись заметно быстрее и стабильнее.

🕷️ ПАРСИНГ: полный append_to_sheet()

services/sheets_export.py — рабочий модуль с комментариями
import gspread
from google.oauth2.service_account import Credentials

SCOPES = ["https://www.googleapis.com/auth/spreadsheets"]


def get_sheet(creds_path: str, spreadsheet_name: str):
    # Загружаем права сервисного аккаунта
    creds = Credentials.from_service_account_file(creds_path, scopes=SCOPES)
    client = gspread.authorize(creds)

    # Открываем таблицу и берём первый лист
    sheet = client.open(spreadsheet_name).sheet1
    return sheet


def append_to_sheet(sheet, rows: list[dict]) -> None:
    if not rows:
        return

    # Преобразуем список словарей в список списков
    prepared = []
    for row in rows:
        prepared.append([
            row.get("title", ""),
            row.get("price", ""),
            row.get("url", "")
        ])

    # Записываем пачкой (быстрее, чем по одной)
    sheet.append_rows(prepared, value_input_option="USER_ENTERED")

Пошагово, что делает модуль

  1. get_sheet — авторизуется сервисным аккаунтом.
  2. Открывает нужную таблицу и берёт лист.
  3. append_to_sheet — готовит данные в формат «список списков».
  4. Пишет всё пачкой для скорости и удобства.
Это уже уровень «клиентский отчёт в облаке», а не просто файл на твоём компьютере.

🧯 ПАРСИНГ: частые ошибки Google Sheets

  • 403 / Permission denied: таблица не расшарена на email сервисного аккаунта.
  • Spreadsheet not found: неверное имя таблицы в client.open(...).
  • Пустая выгрузка: в rows передаётся пустой список.
  • Ломается формат: в словарях нет одинаковых ключей title/price/url.
