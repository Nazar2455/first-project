Сегодня ты учишься делать две ключевые вещи, которые реально продаются: диалоговые формы в боте и умная обработка данных в парсере.

  3. Потом руками — воспроизвести в своём проекте и проверить.

FSM и аналитика кажутся сложными, пока не разбить их на шаги. Сегодня мы именно так и делаем.

🤖 БОТЫ: зачем нужен FSM

FSM (Finite State Machine) — это «машина состояний». Она не даёт пользователю перескочить шаги формы и поддерживает порядок диалога.

Очень простая аналогия

Представь администратора, который вручную заполняет анкету: сначала спрашивает имя, потом телефон, потом комментарий. FSM делает такого администратора «автоматическим», и он никогда не путает этапы.

Что получаем в итоге

  • Пользователь идёт по шагам: имя → телефон → комментарий.
  • Можно отменить процесс командой /cancel.
  • Можно легко добавить валидацию на каждом шаге.

🧪 БОТЫ: мини-примеры FSM (с комментариями)

Пример 1: объявление состояний
from aiogram.fsm.state import State, StatesGroup

# Класс описывает шаги диалога
class LeadForm(StatesGroup):
    name = State()      # Шаг 1: ждём имя
    phone = State()     # Шаг 2: ждём телефон
    comment = State()   # Шаг 3: ждём комментарий
Пример 2: перевод в следующее состояние
@router.message(Command("lead"))
async def start_form(message: Message, state: FSMContext):
    # Переводим пользователя на первый шаг формы
    await state.set_state(LeadForm.name)
    await message.answer("Введите ваше имя")

Смысл: бот «запоминает», на каком шаге сейчас пользователь, и не смешивает этапы.

Мини-пример 3: сохранить промежуточные данные

update_data + get_data
await state.update_data(name="Назар")
data = await state.get_data()
print(data["name"])

Это критично: каждый шаг формы сохраняет кусок данных, а в конце ты собираешь всё как единый объект заявки.

Мини-пример 4: завершить сценарий

Очистка состояния
await state.clear()
await message.answer("Форма завершена")

Если не чистить состояние, пользователь может застрять в старом шаге и бот начнёт отвечать «странно».

🔥 FSM = волшебство диалога
FSM — это не сложное слово. Это когда каждый шаг формы помнит, где ты находишься. Пользователь пишет название → бот запомнил → просит телефон → бот запомнил оба → проверяет. Без FSM бот был бы как блин комом, не запоминал бы контекст. С FSM → это приложение с памятью! 🧠

🤖 БОТЫ: полный сценарий формы

handlers/lead_fsm.py — полный пример с комментариями
from aiogram import Router
from aiogram.filters import Command
from aiogram.types import Message
from aiogram.fsm.context import FSMContext
from aiogram.fsm.state import State, StatesGroup

router = Router()

class LeadForm(StatesGroup):
    name = State()      # Шаг 1: имя
    phone = State()     # Шаг 2: телефон
    comment = State()   # Шаг 3: комментарий

@router.message(Command("lead"))
async def lead_start(message: Message, state: FSMContext):
    # Старт формы — очищаем старые данные на всякий случай
    await state.clear()
    await state.set_state(LeadForm.name)
    await message.answer("📝 Как вас зовут?")

@router.message(Command("cancel"))
async def lead_cancel(message: Message, state: FSMContext):
    # Пользователь может отменить в любой момент
    await state.clear()
    await message.answer("❌ Форма отменена")

@router.message(LeadForm.name)
async def lead_name(message: Message, state: FSMContext):
    # Сохраняем имя в state storage
    await state.update_data(name=message.text.strip())
    # Переходим на следующий шаг
    await state.set_state(LeadForm.phone)
    await message.answer("📞 Введите телефон")

@router.message(LeadForm.phone)
async def lead_phone(message: Message, state: FSMContext):
    text = message.text.strip()

    # Мини-валидация: проверяем, что есть хотя бы 7 цифр
    digits = ''.join(ch for ch in text if ch.isdigit())
    if len(digits) < 7:
        await message.answer("⚠️ Телефон выглядит коротким. Введите корректный номер.")
        return

    await state.update_data(phone=text)
    await state.set_state(LeadForm.comment)
    await message.answer("💬 Добавьте комментарий к заявке")

@router.message(LeadForm.comment)
async def lead_comment(message: Message, state: FSMContext):
    await state.update_data(comment=message.text.strip())

    # Забираем все накопленные данные формы
    data = await state.get_data()

    # Здесь в День 4 подключим сохранение в БД
    await message.answer(
        "✅ Заявка принята!\n"
        f"Имя: {data['name']}\n"
        f"Телефон: {data['phone']}\n"
        f"Комментарий: {data['comment']}"
    )

    # Завершаем FSM-сессию
    await state.clear()

Пошаговая карта полного сценария

  1. /lead — старт, очистка старых данных.
  2. LeadForm.name — сохраняем имя и двигаемся дальше.
  3. LeadForm.phone — проверяем телефон, не пропускаем мусор.
  4. LeadForm.comment — получаем финальные данные и показываем итог.
  5. state.clear() — закрываем сценарий правильно.
Практическая польза: это уже почти готовая форма заявок для реального клиента.

🧯 БОТЫ: частые ошибки FSM и быстрый фикс

  • Обработчик «молчит»: состояние не установлено или не совпадает фильтр.
  • Форма не сбрасывается: забыли await state.clear() в конце.
  • /cancel не работает: обработчик есть, но router не подключён в main.py.
  • Данные теряются: пропущен await state.update_data(...) на одном из шагов.
Мини-практика: добавь временные print/log в каждый шаг FSM — сразу увидишь, где цепочка ломается.

🕷️ ПАРСИНГ: зачем фильтры и TOP-10

Сырые данные редко нужны клиенту. Ему нужен «результат по критериям»: диапазон цены, релевантность, удобная сводка.

Проще: клиент хочет не «весь интернет», а короткий, понятный, применимый список.

Что будет после Дня 3

  • Фильтрация объектов по цене.
  • Вычисление TOP-10 (например, самых дешёвых).
  • Отдельный экспорт отфильтрованных данных.

🧪 ПАРСИНГ: маленькие примеры фильтрации

Пример 1: очистка цены из строки
def parse_price(raw: str) -> int:
    # Оставляем только цифры, убирая валюту/пробелы
    digits = ''.join(ch for ch in raw if ch.isdigit())
    return int(digits) if digits else 0
Пример 2: фильтр по диапазону
def filter_by_price(rows: list[dict], min_price: int, max_price: int) -> list[dict]:
    result = []
    for row in rows:
        price = parse_price(row.get("price", "0"))
        if min_price <= price <= max_price:
            result.append(row)
    return result

Мини-пример 3: сортировка по цене

sorted + key
rows = [
    {"title": "A", "price_num": 400},
    {"title": "B", "price_num": 150},
]

rows_sorted = sorted(rows, key=lambda r: r["price_num"])
print(rows_sorted[0]["title"])  # B

Это основа TOP-10: сначала сортировка, потом берём первые N элементов.

# если ключа нет, код не упадёт

Когда сайты меняют верстку, такие мелочи спасают от падений в проде.

💡 TOP-10 = минимум для клиента, максимум для дохода
Знаешь, за что платят деньги? НЕ за 1000 строк сырых данных. А за 10 лучших вариантов для быстрого решения. Клиент хочет: top_10 отсортировать = готово top_10 дешевых = продал top_10 дорогих = продал Один парсер, разные фильтры = разные проекты, разные деньги 💰

🕷️ ПАРСИНГ: полный модуль фильтра и TOP-10

services/analytics.py — рабочий пример с комментариями
import csv


def parse_price(raw: str) -> int:
    # Преобразуем "12 990 ₽" -> 12990
    digits = ''.join(ch for ch in str(raw) if ch.isdigit())
    return int(digits) if digits else 0


def filter_by_price(rows: list[dict], min_price: int = 0, max_price: int = 10**9) -> list[dict]:
    filtered = []
    for row in rows:
        # Считаем числовую цену из текстового поля
        p = parse_price(row.get("price", "0"))
        if min_price <= p <= max_price:
            # Сохраняем цену в отдельное поле для сортировки
            row["price_num"] = p
            filtered.append(row)
    return filtered


def top_cheapest(rows: list[dict], limit: int = 10) -> list[dict]:
    # Сортируем по числовой цене по возрастанию
    sorted_rows = sorted(rows, key=lambda x: x.get("price_num", 10**9))
    return sorted_rows[:limit]


def export_csv(rows: list[dict], file_path: str):
    if not rows:
        return

    # Берём заголовки по ключам первого объекта
    fieldnames = list(rows[0].keys())
    with open(file_path, "w", encoding="utf-8", newline="") as f:
        writer = csv.DictWriter(f, fieldnames=fieldnames)
        writer.writeheader()
        writer.writerows(rows)

Пошаговый разбор логики модуля

  1. parse_price — превращает «грязную» цену в число.
  2. filter_by_price — отсекает всё вне диапазона.
  3. top_cheapest — сортирует и возвращает первые N.
  4. export_csv — сохраняет результат в читаемый отчёт.
Это уже мини-пайплайн аналитики, который можно продавать как «обработку и ранжирование данных».

🧯 ПАРСИНГ: частые ошибки аналитики

  • TOP-10 пустой: до сортировки пришёл пустой список (проверь фильтр).
  • Сортировка странная: сортируешь строковые цены, а не числовые.
  • CSV не открывается корректно: пропущены encoding="utf-8" и newline="".
  • Поля «прыгают»: у объектов разные ключи, из-за этого ломается единый формат экспорта.
