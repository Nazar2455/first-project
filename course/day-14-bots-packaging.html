1) Каркас кейса

  • Проблема клиента (до).
  • Что реализовано (функции бота).
  • Результат (после).
  • Срок/стоимость MVP.

2) Шаблон блока «Для кого подходит»

README / портфолио
Для кого:
- малый бизнес, который теряет заявки в мессенджерах
- проекты с ручной обработкой входящих сообщений
- команды, где нужен экспорт лидов в таблицу

3) Публикация и дистрибуция

  1. Обнови GitHub README + release.
  2. Добавь кейс в профиль фриланс-площадки.
  3. Сделай 3 поста в тематических чатах/каналах.
Без публикации кейс не продаёт — он просто лежит в репозитории.

4) Практика KPI

  1. Упакуй 2 бот-кейса.
  2. На каждый — 1 демо-видео + 2 скриншота.
  3. Сделай минимум 10 откликов с новыми кейсами.
  4. Замерь, где ответы приходят быстрее.
