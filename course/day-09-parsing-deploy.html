День 9 · 🕷️ ПАРСИНГ

Тема: стабильный запуск парсера по расписанию и контроль, что отчёты приходят без ручного запуска.

← День 9 (выбор трека) Переключиться на БОТЫ День 10 →

1) Архитектура «парсер в проде»

  • Сценарий main.py выполняется без ручного ввода.
  • Логи пишутся в файл или консоль хостинга.
  • Ошибки отлавливаются и не ломают весь цикл.
  • Результаты сохраняются в CSV/JSON/Sheets.

2) Пример логирования запуска

logging setup
import logging

logging.basicConfig(
    level=logging.INFO,
    format="%(asctime)s [%(levelname)s] %(message)s"
)

logging.info("Parser started")
# ... парсинг
logging.info("Parser finished successfully")

3) Расписание (Windows и Linux)

Windows Task Scheduler

  • Создай задачу 2 раза в день.
  • Укажи рабочую директорию проекта.
  • Включи логирование stdout/stderr.

Linux cron

crontab
# Запуск в 09:00 и 18:00
0 9,18 * * * /usr/bin/python3 /path/to/parser/main.py >> /path/to/parser.log 2>&1
Сначала отлаживаем ручной запуск, потом включаем расписание.

4) Практика

  1. Настрой автозапуск парсера 1–2 раза в день.
  2. Проверь логи после автоматического запуска.
  3. Убедись, что результат обновился без твоего участия.
  4. Добавь в README раздел «Как запустить по расписанию».
