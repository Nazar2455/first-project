Практика дня 13

🤖 БОТЫ: bot-template

Цель: превратить набор файлов в переиспользуемый шаблон, с которым новый бот стартует быстрее и стабильнее.

Связь с теорией дня 13: закрепляешь слоистую архитектуру, единую сборку приложения и fail-fast конфиг.

Фокус на смысл: шаблон нужен, чтобы экономить время на каждом новом проекте и снижать количество повторяющихся ошибок.

Что нужно сделать (конкретно)

  1. Есть слои handlers/services/repositories/utils.
  2. Единая сборка через create_app + load_settings.
  3. Fail-fast по TELEGRAM_BOT_TOKEN.
  4. Минимальные команды /start, /health, admin guard.

Критерии готовности (Definition of Done)

  • Структура шаблона понятна и повторяемо запускается.
  • При отсутствии токена запуск падает сразу с понятным сообщением.
  • Базовый набор команд и guard работают из коробки.

Что сдать на проверку

  • Код шаблона с create_app/load_settings.
  • Лог запуска и короткий замер bootstrap-time.

🕷️ ПАРСИНГ: parser-template

Цель: собрать parser-template, который подключает новые источники через адаптер, не ломая ядро пайплайна.

Связь с теорией дня 13: закрепляешь BaseAdapter, run_pipeline и quality-report как стандарт для любых ниш.

Фокус на смысл: новый сайт = новый адаптер, а не переписывание всего проекта.

Что нужно сделать (конкретно)

  1. Есть BaseAdapter + SiteAdapter с selectors.
  2. Pipeline: fetch → parse → validate → export.
  3. Есть quality-report total/valid/skipped/time.
  4. Экспорт в CSV/JSON и переиспользуемый run_pipeline.

Критерии готовности (Definition of Done)

  • Адаптерный контракт соблюдается и расширяется без хаоса.
  • Метрики качества выводятся после каждого запуска.
  • Файлы выгрузки формируются стабильно в одном формате.

Что сдать на проверку

  • Код BaseAdapter/SiteAdapter + run_pipeline.
  • Лог прогона с total/valid/skipped/time.
