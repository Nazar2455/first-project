🕷️ ПАРСИНГ: parsing-brief + objection matrix

Цель: подготовить продажную заготовку под парсинг-проект с чёткими SLA и обработкой типовых возражений.

Связь с теорией дня 19: переводишь техничный оффер в коммерчески понятный формат для отклика и переговоров.

Фокус на смысл: клиент покупает не «парсер», а предсказуемый результат по частоте, качеству и срокам.

Что нужно сделать (конкретно)

  1. Есть source/fields/frequency/output/SLA.
  2. Есть render_parsing_brief.
  3. Есть матрица возражений.
  4. Есть short/full формат для отклика.

Критерии готовности (Definition of Done)

  • Brief можно отправить клиенту без дополнительной расшифровки.
  • На 2–3 ключевых возражения есть готовые ответы.
  • Есть версия сообщения для чата и для длинного документа.
