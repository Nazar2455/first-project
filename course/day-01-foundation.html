Модуль 1 · День 1: фундамент для заработка

Главная мысль: твоя цель сегодня — не «выучить всё в теории», а создать рабочий фундамент двух направлений: Telegram-бот и парсер. Это твоя первая производственная база.

Мы идём по принципу: понял → сделал → проверил → сохранил результат. Такой подход делает тебя не «читателем статей», а человеком, который реально выполняет заказы.

🧠 Как думать в первый день (и почему это важно)

Новички часто пытаются «сначала разобраться вообще во всём». Из-за этого теряется темп и мотивация. Профессиональный подход другой:

  1. Сначала получить минимальный рабочий результат (MVP).
  2. Потом улучшать структуру и качество.
  3. Параллельно учиться, но через практику.
Сегодня ты строишь фундамент. Фундамент в доходных проектах — это: структура папок, управление зависимостями, запуск приложения и контроль ошибок.

Что такое «рабочее окружение»

Окружение — это отдельная мини-среда Python для конкретного проекта. Если не изолировать окружения, библиотеки начнут конфликтовать между проектами.

Правило специалиста: 1 проект = 1 виртуальное окружение.

🐍 Python, venv, requirements, .env — разжёвано

1) Зачем нужен requirements.txt

Это список зависимостей с версиями. Без фиксации версий проект может «вдруг сломаться» из-за обновлений библиотек.

2) Зачем нужен .env

В .env хранятся секреты: токены, ключи, ID админов. Секреты нельзя оставлять в коде и нельзя пушить в GitHub.

Пример .env
TELEGRAM_BOT_TOKEN=your_token_here
ADMIN_IDS=123456789

3) Зачем нужен .env.example

Это шаблон без секретов, чтобы другой разработчик понимал, какие переменные нужны проекту.

.env.example
TELEGRAM_BOT_TOKEN=
ADMIN_IDS=

🔍 Как читать код, если ты не робот

Ты дал супер-точное замечание: «код есть, но как его читать?». Вот универсальная схема чтения, которой пользуются джуны и мидлы.

  1. Шаг 1 — найди вход: где начинается выполнение? Обычно это main() или блок if __name__ == "__main__".
  2. Шаг 2 — найди данные: что приходит на вход и что выходит на выход.
  3. Шаг 3 — найди узкие места: где могут быть ошибки (сеть, файл, внешний API).
  4. Шаг 4 — проследи путь: по одной функции сверху вниз, не прыгая хаотично.
Правило чтения: сначала понимаем смысл блока на уровне «что делает», и только потом углубляемся в синтаксис.

Мини-пример 1: функция

Простейшая функция

    return f"Привет, {name}!"

Разбор: def — объявление функции, name: str — ожидаем строку, -> str — функция возвращает строку.

try/except

    x = int("123")

    x = 0

Разбор: если преобразование к числу не удалось — приложение не падает, а переходит в безопасный сценарий.

Минимальный рабочий код

main.py
import asyncio
import os
from aiogram import Bot, Dispatcher
from aiogram.filters import Command
from aiogram.types import Message
from dotenv import load_dotenv

load_dotenv()
TOKEN = os.getenv("TELEGRAM_BOT_TOKEN")

if not TOKEN:
    raise ValueError("Не найден TELEGRAM_BOT_TOKEN в .env")

dp = Dispatcher()

@dp.message(Command("start"))
async def cmd_start(message: Message):
    await message.answer("Привет! Я твой lead-бот. Напиши /help")

@dp.message(Command("help"))
async def cmd_help(message: Message):
    await message.answer("Доступные команды: /start, /help")

async def main() -> None:
    bot = Bot(token=TOKEN)
    await dp.start_polling(bot)

if __name__ == "__main__":
    asyncio.run(main())

Построчный разбор

  • load_dotenv() — загружает переменные из файла.
  • TOKEN = os.getenv(...) — берём токен из окружения.
  • Проверка if not TOKEN — ранний контроль ошибки.
  • @dp.message(Command("start")) — хендлер команды.
  • start_polling — бесконечный цикл обработки сообщений.
Почему это продаваемо: ты уже умеешь поднять базовый бот-сервис, который можно расширить до формы заявок, FAQ, CRM-функций.

Код стартового парсера

main.py
import csv
import requests
from bs4 import BeautifulSoup

URL = "https://example.com/catalog"


def fetch_html(url: str) -> str:
    response = requests.get(url, timeout=20)
    response.raise_for_status()
    return response.text


def parse_cards(html: str) -> list[dict]:
    soup = BeautifulSoup(html, "html.parser")
    cards = soup.select(".product-card")
    rows = []

    for card in cards[:100]:
        title_el = card.select_one(".title")
        price_el = card.select_one(".price")
        if not title_el or not price_el:
            continue

        rows.append({
            "title": title_el.get_text(strip=True),
            "price": price_el.get_text(strip=True)
        })
    return rows


def save_csv(rows: list[dict], file_path: str = "result.csv") -> None:
    with open(file_path, "w", encoding="utf-8", newline="") as f:
        writer = csv.DictWriter(f, fieldnames=["title", "price"])
        writer.writeheader()
        writer.writerows(rows)


def main() -> None:
    html = fetch_html(URL)
    rows = parse_cards(html)
    save_csv(rows)
    print(f"✅ Сохранено строк: {len(rows)}")


if __name__ == "__main__":
    main()

Что здесь профессионально

  • Код разбит на функции: загрузка, парсинг, сохранение.
  • Есть обработка HTTP-ошибок через raise_for_status().
  • Логика легко тестируется и масштабируется.
Для фриланса важен не «хитрый» код, а надёжность и предсказуемость результата.

🧯 Частые ошибки Дня 1 и быстрый ремонт

Бот не запускается

  • Проверь токен в .env.
  • Убедись, что в проекте активировано окружение.
  • Проверь, что установлены зависимости из requirements.txt.

Парсер вернул 0 объектов

  • Скорее всего, CSS-селектор не совпадает с реальной HTML-структурой.
  • Открой страницу в браузере и перепроверь классы.
  • Проверь, не грузится ли контент через JavaScript (в таком случае нужен другой подход).

CSV «кривой» или с кракозябрами

  • Используй encoding="utf-8".
  • Не забывай newline="" при записи CSV в Windows.
