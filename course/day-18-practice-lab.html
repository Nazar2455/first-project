🤖 БОТЫ: mini-CRM статусы

Цель: сделать управляемый цикл обработки лидов через статусы и фильтры.

Связь с теорией дня 18: отрабатываешь логику CRM-пайплайна: валидация переходов, контроль стадий и отчётность по лидам.

Фокус на смысл: CRM полезна только если ты быстро видишь, где лид «застрял», и можешь безопасно перевести его на следующий шаг.

Что нужно сделать (конкретно)

  1. Есть ALLOWED_STATUSES и валидация переходов.
  2. Есть list/filter лидов по статусу.
  3. Есть set_lead_status с проверкой id.
  4. Есть базовый workflow new → in_work → done.

Критерии готовности (Definition of Done)

  • Некорректный статус или id не ломает обработку.
  • Можно получить лидов по каждому статусу отдельно.
  • В логах видна история изменений статусов.

🕷️ ПАРСИНГ: админ-дашборд

Цель: вывести операционную панель по источникам парсинга с быстрыми действиями.

Связь с теорией дня 18: закрепляешь observability: состояние по источникам, KPI и ручные операционные команды.

Фокус на смысл: дашборд должен помогать принимать решение за секунды: где всё ок, где деградация и что перезапустить.

Что нужно сделать (конкретно)

  1. Есть PARSER_STATE по источникам.
  2. Есть dashboard text/report builder.
  3. Есть restart/операционная команда.
  4. Есть статусы ok/warning/error.

Критерии готовности (Definition of Done)

  • Панель показывает состояние минимум по двум источникам.
  • Есть явные триггеры warning/error.
  • Операция restart оставляет понятный след в логе.
