1) Что хочет увидеть клиент в кейсе бота

  • Проблема: «лиды теряются, заявки неструктурированы».
  • Решение: «бот собирает, хранит, экспортирует, уведомляет».
  • Результат: экономия времени и прозрачность заявок.
Формула кейса: Проблема → Что сделал → Что получил клиент.

2) Шаблон продающего README

README.md (каркас)
# Lead Bot for Telegram

## Что решает
Бот автоматизирует сбор заявок из Telegram и сохраняет их в базу.

## Функции
- /start, /help
- форма заявки (FSM)
- админ-команды /admin и /export
- экспорт лидов в CSV

## Стек
Python 3.11, aiogram 3, aiosqlite, python-dotenv

## Быстрый запуск
1. Создать .env
2. Установить зависимости: pip install -r requirements.txt
3. Запустить: python main.py

## Результат для бизнеса
- меньше ручной рутины
- выше скорость ответа
- прозрачная история заявок

3) Сценарий видео-демо (60–90 сек)

  1. 5 сек: название проекта + что решает.
  2. 25 сек: путь пользователя (/start → форма → подтверждение).
  3. 20 сек: путь администратора (/admin → /export).
  4. 10 сек: итог в CSV + выгода для клиента.
  5. 5 сек: «готов адаптировать под ваш сценарий».
