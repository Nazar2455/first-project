1) Минимум для облачного запуска

  • requirements.txt с фиксированными версиями.
  • Точка входа: python main.py.
  • Токен и admin IDs в переменных окружения.
  • Логирование старта и критических ошибок.
Если бот не запускается в облаке, в 80% случаев проблема в env-переменных или зависимостях.

2) Пример startup-логов

main.py (идея)
import asyncio
import logging

logging.basicConfig(level=logging.INFO)

async def main():
    logging.info("Bot is starting...")
    # здесь запуск dp.start_polling(bot)


if __name__ == "__main__":
    try:
        asyncio.run(main())
    except Exception as e:
        logging.exception("Fatal error during startup: %s", e)
        raise

3) Чеклист «бот в проде»

  1. После деплоя команда /start отвечает.
  2. FSM форма проходит полный цикл без ошибок.
  3. /admin доступен только админам.
  4. В логах нет постоянных ошибок/restart loop.
