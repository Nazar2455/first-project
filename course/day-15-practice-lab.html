🤖 БОТЫ: quality-gate шаблона

Цель: автоматизировать базовую проверку качества перед релизом шаблона бота.

Связь с теорией дня 15: закрепляешь quality-gate подход и обязательные проверки релизной готовности.

Фокус на смысл: релиз должен блокироваться автоматически, если отсутствуют критичные части (env, guard, логирование, quickstart).

Что нужно сделать (конкретно)

  1. Есть required-checks контракт.
  2. Есть missing список и блокировка релиза.
  3. Есть позитивная ветка «можно выпускать».
  4. Покрыты env, команды, admin-guard, error-logging, README.

Критерии готовности (Definition of Done)

  • При неполном шаблоне релиз останавливается с понятным списком missing.
  • При полном шаблоне quality-gate даёт «ready» сигнал.
  • Есть проверяемый output для smoke-check.

Что сдать на проверку

  • Код quality-gate и список required-checks.
  • Два примера output: fail и pass.

🕷️ ПАРСИНГ: parser-template v2 multi-site

Цель: запускать несколько источников единым раннером и получать единый quality-report по каждому сайту.

Связь с теорией дня 15: закрепляешь реестр адаптеров, run_all orchestration и единый формат отчётов.

Фокус на смысл: multi-site режим должен масштабировать парсинг без копирования логики по файлам.

Что нужно сделать (конкретно)

  1. Есть BaseAdapter + validate_rows.
  2. Есть run_adapter с report dict.
  3. Есть run_all/adapters registry для 2+ сайтов.
  4. Есть метрики total/valid/skipped/source.

Критерии готовности (Definition of Done)

  • Раннер отрабатывает минимум 2 адаптера подряд.
  • По каждому источнику есть quality-метрики в одном формате.
  • Сбой одного источника не ломает отчёт по остальным (или корректно фиксируется).

Что сдать на проверку

  • Код registry + run_all + run_adapter.
  • Логи multi-site прогона по минимум 2 источникам.
