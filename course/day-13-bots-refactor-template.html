1) Целевая структура шаблона

bot-template
bot-template/
├── handlers/
├── services/
├── db/
├── states/
├── utils/
├── main.py
├── requirements.txt
├── .env.example
└── README.md

2) Принцип распределения кода

  • handlers: только входящие команды/сообщения.
  • services: бизнес-логика.
  • db: только операции с БД.
  • utils: вспомогательные функции (security, logging).
Если в handler 200+ строк — это не handler, это сигнал к рефакторингу.

3) Мини-шаблон инициализации

main.py (идея)
from aiogram import Bot, Dispatcher
from handlers import common, admin


def build_dispatcher() -> Dispatcher:
    dp = Dispatcher()
    dp.include_router(common.router)
    dp.include_router(admin.router)
    return dp

4) Практика

  1. Перенеси текущий бот в структуру шаблона.
  2. Проверь, что /start, FSM и /admin всё ещё работают.
  3. Создай branch refactor-commercial.
  4. Подготовь readme «как стартовать новый бот из шаблона».
