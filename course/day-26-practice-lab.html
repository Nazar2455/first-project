🤖 БОТЫ: support packages

Цель: разработать подписочную линейку поддержки бота с прозрачными SLA и ограничениями.

Связь с теорией дня 26: закрепляешь recurring-модель: уровни сервиса, время реакции, фиксы и рамки изменений.

Фокус на смысл: хороший SLA-пакет снижает неопределённость: клиент понимает, когда получит реакцию и что входит в обслуживание.

Что нужно сделать (конкретно)

  1. Есть tiers lite/standard/pro.
  2. Есть response/fix SLA.
  3. Есть limits changes/month.
  4. Есть build_support_offer.

Критерии готовности (Definition of Done)

  • Пакеты различаются по скорости и объёму сервиса.
  • Есть явная ценовая модель.
  • Оффер формируется как готовый текст для клиента.

Что сдать на проверку

  • Код SLA-пакетов.
  • Черновик оффера по каждому тарифу.

🕷️ ПАРСИНГ: retainer + monthly report

Цель: оформить подписку на поддержку парсинга с регулярной отчётностью и SLA.

Связь с теорией дня 26: строишь сервисную модель сопровождения: мониторинг, алерты, адаптация и ежемесячные отчёты.

Фокус на смысл: клиент покупает предсказуемость: стабильный поток данных + понятные отчёты по качеству.

Что нужно сделать (конкретно)

  1. Есть parsing retainer SLA.
  2. Есть monthly report format.
  3. Есть includes (monitoring/alerts/adaptation).
  4. Есть генератор коммерческого сообщения.

Критерии готовности (Definition of Done)

  • В отчёте есть метрики качества и сбоев.
  • Сервисный пакет чётко описывает включённые работы.
  • Коммерческий текст готов к отправке клиенту.
