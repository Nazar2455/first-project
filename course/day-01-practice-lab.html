🤖 Практическое задание #1 (БОТЫ)

Цель: собрать рабочий каркас Telegram-бота (MVP), который безопасно запускается из .env и отвечает на базовые команды /start, /help, /about.

Связь с теорией дня 1: здесь ты закрепляешь три опоры из теории — окружение проекта (зависимости), конфиг через .env (без хардкода токена) и базовую архитектуру aiogram (Bot + Dispatcher + handlers).

Что нужно сделать (конкретно)

  1. Подготовь окружение: установи aiogram и python-dotenv, создай .env с переменной TELEGRAM_BOT_TOKEN.
  2. В коде загрузи токен через load_dotenv() + os.getenv() и добавь проверку: если токен пустой — бот не должен молча запускаться.
  3. Создай 3 отдельных хендлера:
    /start — приветствие и что умеет бот;
    /help — список команд;
    /about — кратко о назначении бота.
  4. Собери async main() и запуск через asyncio.run(main()), чтобы бот стабильно переходил в polling.

Критерии готовности (Definition of Done)

  • Бот реально отвечает в Telegram минимум на 3 команды.
  • Токен не захардкожен в коде, а берётся из .env.
  • При пустом токене есть понятная ошибка (а не «тихое» падение/зависание).
Подсказка 1: какие импорты должны быть

Минимум: asyncio, os, Bot, Dispatcher, Command, Message, load_dotenv.

🕷️ Практическое задание #2 (ПАРСИНГ)

Цель: собрать первый предсказуемый pipeline парсинга: получить HTML, безопасно распарсить карточки, отфильтровать мусор и сохранить результат в CSV.

Связь с теорией дня 1: это практика по цепочке «входные данные → обработка → выходной артефакт». Ты тренируешь базовую инженерную дисциплину: сетевой запрос, парсинг, валидация, экспорт.

Что нужно сделать (конкретно)

  1. Раздели код на 4 функции: fetch_html(), parse_cards(), save_csv(), main() (каждая отвечает за свой шаг).
  2. В fetch_html() используй requests.get(..., timeout=...) и raise_for_status(), чтобы не игнорировать HTTP-ошибки.
  3. В parse_cards() через BeautifulSoup и select/select_one вытащи минимум поля title и price (лучше + url).
  4. Добавь базовую валидацию: если карточка пустая/битая — пропускай её.
  5. Сохрани результат в CSV (UTF-8, newline=""), ограничь выгрузку первыми 100 строками и выведи в консоль итог: сколько строк сохранено.

Критерии готовности (Definition of Done)

  • Скрипт запускается через main() без ручных правок «на лету».
  • На выходе появляется CSV-файл с заголовками и данными.
  • В консоли есть итоговая метрика (например, Сохранено строк: 100).
Подсказка 1: минимальная защита от падений

Перед чтением текста из элемента проверяй, что элемент не None.

Подсказка 2: корректная запись CSV на Windows

Открывай файл так: open(path, "w", encoding="utf-8", newline="").
